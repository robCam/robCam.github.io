ES6 Mobile Lazy Load

This is an experimental proof-of-concept project investigating the edge cases associated with fetching data remoteley from a RESTful system, via a mobile device.

The edge cases for a mobile device fetching data remotely via a RESTful API are as follows:

  • No network connection.
  • The RESTful database may not be available.
  • The network is slow (E.g. A GPRS connection is used).

To provide the user with a performant experience this project attempts to incorporate the user-centric performance model known as RAIL conceived by Google Chrome team.

This project is created using HTML5, CSS, and JavaScript (ES6) developed by Rob Campbell, using some of the new ECMAScript 6 features as follows:

  1. Classes
  2. Formal Method definitions
  3. Promises
  4. Arrow Functions
  5. Template Strings
  6. Modules (import and export)
  7. Object.assign() method
  8. const and let declarations

This project depends on a local install of Apache CouchDB for hosting the remote data as the app interfaces with it via its RESTful API. NOTE: At present this is not made publicly available.

JavaScript linting for this project is handled by ESLint and it uses a custom eslintrc.json file.

To transpile ES6 (ES2015) to ES5 the Grunt build for this project utilises Babel, and Browserify (for handling the ES6 Modules via CommonJS).

You'll need to:

  1. Install node.js.
  2. Install npm.
  3. Run 'npm install'.
  4. Run 'grunt' to produce a distribution build, 'grunt dev' for development, and 'grunt docs' to generate documentation
